# Autorité des documents d'architecture

Ce dossier contient les **spécifications d'application** des ADR adoptés dans `decisions/`.

## Ordre d'autorité

1. `CONSTITUTION.md` et ses amendements adoptés ;
2. décisions adoptées dans `decisions/` ;
3. spécifications d'architecture de ce dossier ;
4. contrats UX, design et catalogue exécutable ;
5. code et tests.

Une spécification de ce dossier précise une décision ; elle ne peut ni la remplacer ni la diminuer. En cas de contradiction, la source supérieure s'applique et la contradiction doit être enregistrée.

## Identifiants

Les identifiants `ADR-xxxx` normatifs appartiennent exclusivement aux fichiers adoptés de `decisions/`. Les anciens brouillons ayant porté les mêmes identifiants sont conservés dans `history/architecture-drafts/` comme traces historiques non normatives.

## Statuts

- **Décision adoptée** : texte normatif immuable ; toute évolution passe par une nouvelle décision ou un addendum tracé.
- **Spécification d'application** : traduction maintenue d'une décision adoptée.
- **Proposition** : hypothèse non autorisée pour l'implémentation tant qu'elle n'est pas validée.
